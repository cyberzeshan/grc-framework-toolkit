## 1. Program Overview

### 1.1 Purpose

This Information Security Program Charter formally establishes the **[Organization Name]** Information Security Program ("the Program"). It defines the Program's mandate, authority, scope, objectives, and governance structure, and secures organizational commitment to resource and operate the Program on an ongoing basis.

Cybersecurity risk is a top-tier organizational risk. This charter signals that **[Organization Name]** leadership accepts accountability for managing that risk and has committed to the people, processes, and technology required to do so effectively.

### 1.2 Mission Statement

> *[Organization Name]'s Information Security Program exists to protect the confidentiality, integrity, and availability of the organization's information assets — enabling the business to operate with confidence, meet its obligations to customers and regulators, and grow without introducing unacceptable risk.*

### 1.3 Vision

> *[One sentence describing the long-term security posture the organization aspires to — e.g., "To be a Tier 3 organization on the NIST Cybersecurity Framework within 24 months, capable of detecting, responding to, and recovering from cybersecurity threats without material impact to operations or reputation."]*

## 2. Authority and Mandate

### 2.1 Program Authority

This charter grants the Information Security Program the following authorities, exercised through the CISO and the security team:

| Authority | Description |
|:---|:---|
| **Policy Authority** | Develop, publish, and enforce information security policies across all business units |
| **Risk Authority** | Identify, assess, and require treatment of information security risks; escalate unaccepted risks to the executive team |
| **Audit Authority** | Audit compliance with security policies and controls; engage third-party assessors |
| **Veto Authority** | Delay or halt deployment of systems, products, or vendor integrations that present unacceptable security risk pending remediation |
| **Access Authority** | Access all systems, logs, and data required to perform security assessments and incident investigations |
| **Vendor Authority** | Define and enforce security requirements for all vendors with access to organizational systems or data |

### 2.2 Executive Mandate

The following executives mandate the establishment and operation of the Information Security Program and commit to its resourcing and support:

| Executive | Title | Mandate |
|:---|:---|:---|
| [Name] | Chief Executive Officer | Ultimate accountability for organizational cyber risk |
| [Name] | Chief Information Officer | Ensures IT infrastructure supports security requirements |
| [Name] | Chief Financial Officer | Allocates budget to support Program objectives |
| [Name] | General Counsel | Ensures regulatory compliance obligations are met |
| [Name] | Chief People Officer | Enforces security requirements in HR processes |
| [Name] | VP Engineering | Ensures secure software development practices |

### 2.3 Board Oversight

The Board of Directors (or Audit/Risk Committee) receives the following:

- **Quarterly:** Cybersecurity risk briefing from the CISO or security program owner
- **Annually:** Full Program review including posture assessment, incidents, and roadmap
- **Ad Hoc:** Immediate notification of material cybersecurity incidents or significant risk changes

## 3. Scope

### 3.1 Organizational Scope

This Program applies to **all [Organization Name] personnel**, including:

- Full-time and part-time employees
- Contractors and consultants
- Temporary workers and interns
- Board members with system access

### 3.2 Systems and Asset Scope

This Program covers all information assets owned, operated, or controlled by [Organization Name], including:

- All hardware: laptops, servers, mobile devices, IoT devices
- All software: SaaS applications, custom-built systems, open source components
- All data: customer data, employee data, financial records, intellectual property
- All cloud environments: production, staging, and development
- All physical locations: headquarters, remote offices, data centers
- All third-party systems that access, process, or store organizational data

## 5. Governance Structure

### 5.1 Roles and Responsibilities (RACI)

| Responsibility | CEO | CISO | CIO | GC | CPO | BU Leads | All Staff |
|:---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| Approve security strategy and budget | A | R | C | C | C | I | |
| Define risk appetite | A | R | C | C | | C | |
| Develop and maintain security policies | I | A/R | C | C | C | C | |
| Enforce security policies | I | A | C | C | C | R | R |
| Conduct risk assessments | I | A/R | C | | | C | |
| Implement technical security controls | I | A | R | | | R | |
| Complete security awareness training | I | A | I | | R | R | R |
| Report security incidents | I | A | C | C | | R | R |
| Manage vendor security | I | A | C | C | | R | |
| Report to board | I | R | C | C | | | |

*R = Responsible | A = Accountable | C = Consulted | I = Informed*

### 5.2 Security Governance Committee

A Security Governance Committee ("the Committee") is established with the following membership and meeting cadence:

| Member | Title | Role in Committee |
|:---|:---|:---|
| [Name] | CEO | Executive Sponsor; chairs annual full review |
| [Name] | CISO | Leads Committee; presents agenda |
| [Name] | CIO | IT risk and infrastructure representation |
| [Name] | General Counsel | Legal and regulatory representation |
| [Name] | CFO | Budget and financial risk representation |
| [Name] | VP Engineering | Engineering and product security |
| [Name] | VP Operations | Business continuity representation |

**Meeting Cadence:**
- Monthly: Security operations review (CISO + CIO)
- Quarterly: Committee meeting — risk review, metrics, incidents, roadmap
- Annually: Full board presentation and program review

### 5.3 CISO Reporting Line

The CISO reports to: **[CEO / CTO / Board / Other — choose one and justify]**

> **Note:** The CISO reporting line has significant implications for program effectiveness and independence. Best practice (and an ISO 27001 expectation) is for the CISO to report to the CEO or Board, not to the CIO, to ensure independence of security from IT operations.

## 6. Program Components

The following components constitute the Information Security Program:

| Component | Description | Owner |
|:---|:---|:---|
| **Risk Management** | Annual risk assessment, risk register, risk treatment planning | CISO |
| **Policy Management** | Security policy suite development, publication, review, and enforcement | CISO |
| **Asset Management** | Hardware, software, and data asset inventory and lifecycle management | IT / CISO |
| **Identity and Access Management** | User provisioning, MFA, PAM, access reviews | IT / CISO |
| **Vulnerability Management** | Scanning, patching, and remediation tracking | Security / IT |
| **Security Awareness Training** | Annual training for all staff; role-specific training | HR / CISO |
| **Threat Detection and Monitoring** | SIEM, log management, security alerting | Security Operations |
| **Incident Response** | IRP, CSIRT, tabletop exercises, post-incident reviews | CISO / Security |
| **Business Continuity** | BCP/DRP integration with information security | BCP Owner / CISO |
| **Vendor Risk Management** | Third-party security assessments, contract requirements | Procurement / CISO |
| **Compliance and Audit** | Regulatory compliance tracking, internal and external audits | CISO / Legal |
| **Secure Development** | Secure SDLC, code review, SAST/DAST, developer training | Engineering / CISO |
| **Physical Security** | Physical access controls, secure area management | Facilities / IT |
